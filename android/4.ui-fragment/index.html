UI fragment与fragment管理器

fragment是一种控制器对象，管理用户界面的fragment又称为UI fragment，它自己也有产生于布局文件xml的视图，activity的视图能预留位置供fragment视图插入，便于绕开activity使用规则的限制（例如界面运行时的组装和切换等），此时称该activity托管了该fragment实例。

本章节内容以CriminalIntent应用为例。

新的项目

CriminalIntent应用涉及的对象：

  1. 模型层

① Crime类：实例代表一种办公室陋习，有标识ID作为识别Crime实例的唯一元素

② CrimeLab类：数据集中存储池，用来存储Crime对象

  1. 控制器层

①CrimeActivity类：其视图由FrameLayout组件组成，用来显示CrimeFragment视图

②CrimeFragment类：其视图由多个组件组成，并设有监听器以响应用户操作

③CrimeListActivity类：其视图由FrameLayout组件组成，用来显示CrimeListFragment视图

④CrimeListFragment类：其视图由RecyclerView组成，用于显示Crime列表

创建Crime类

UUID是Android框架里的Java工具类，调用UUID.randomUUID()可以方便的产生一个随机唯一的值：

/* Crime.java */

public class Crime{
    private UUID mId;
    private String mTitle;
    private Date mDate;   //java.util.Date类
    private boolean mSolved;
    
    public Crime(){
        mId = UUID.randomUUID();
        mDate = new Date();
    }
}

在构造方法下的空白处，==右键→Generate→Getter/Getter and Setter==快捷生成需要的方法。

创建CrimeActivity类

FrameLayout

此Activity托管着UI fragment，在视图层级结构中为fragment视图安排位置：

<!-- activity_crime.xml -->

<?xml version="1.0" encoding="utf-8"?>
<FrameLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/fragment_container"
    android:layout_height="match_parent"
    android:layout_width="match_parent"/>

FrameLayout是服务于CrimeFragment的容器视图，但也可以用于托管其他的fragment。

FragmentManager

FragmentManager类管理：

  • fragment队列
  • fragment事务回退栈（见……

引入Fragment类时在CrimeActivity类添加FragmentManager类以协同工作。要以代码的方式将fragment添加给activity，需要直接调用activity的FragmentManager：

/* CrimeActivity.jaba */
public class CrimeActivity extends AppCompatActivity{
    
    @Override
    protected void onCreate(Bundle savedInstanceState){
        super.onCreate(savedInstanceState);
        setContextView(R.layout.activity_crime);  //显示视图
        
        /*获取FM*/
        FragmentManager fm = getSupportFragmentManager();
        /*获取fragment并把它交给fm管理*/
        /*fm使用FrameLayout组件的资源ID（容器视图资源ID）来识别CrimeManager（UI fragment）*/
        Fragment fragment = fm.findFragmentById(R.id.fragment_container);
               
        if (fragment == null){//如果要获取的fragment不在队列中
            fragment = new CrimeFragment();
            fm.beginTransaction()  //创建并返回FragmentTransaction实例，支持流接口的链式方法调用
                .add(R.id.fragment_container,fragment)  //添加fragment，告诉fm，fragment视图出现在activity的位置，并唯一标识fm队列中的fragment
                .commit();     //提交fragment事务
        }
    }
    
}

托管UI fragment

fragment在API 11级系统版本中引入，现在有原生版本、支持库版本两种fragment实现可供选择。

原生版本的fragment实现内置在系统设备中，应用在不同设备上运行可能会有不同表现。

支持库版本的fragment在类库里，发布时会打包在应用里。更新方便，轻松升级，但需要占用额外空间。

添加依赖关系

引例使用的支持库版fragment来自于一个名为AppCompat的兼容库，要使用这个支持库需要被项目列入依赖关系，在app/build.gradle文件中设置：

...
dependencies{
    compile fileTree(dir:'libs',include:['*.jar'])
    ...
    compile 'com.android.support:appcompat-v7:25.0.1'
}

//编写项目时 compile已经改成了 implementation，且提示建议迁移到AndroidX库

当前依赖设置如上，表明项目依赖于libs目录下的所有jar包。

应用编译时，Gradle会找到并下载依赖包。如果没有发现AppCompat库，可以在菜单==File→Project Structure→Dependencies==，选择Library dependency界面添加新的依赖项。

生命周期与FragmentManager

fragment的生命周期与activity有相似之处，fragment的状态反映着activity的状态，但与activity生命周期的一个关键区别为其生命周期方法不由操作系统调用。

activity的FragmentManager负责调用队列中fragment的是生命周期方法且接收操作系统的调用指令，让fragment和activity的状态保持一致。

托管

activity托管UI fragment有两种方式：

  1. 在activity布局中添加fragment：即使用布局fragment，将fragment及其视图与activity视图绑定，不灵活
  2. 在activity代码中添加fragment：可以动态控制fragment

使用第二种方法需要在activity视图层级结构中为fragment视图安排位置，在先前配置activity_crime.xml文件时已经完成此操作。

CrimeFragment用于显示Crime实例的信息，编写布局文件与编写activity布局文件方法一致，在此不赘述。

其中定义了如下组件的资源ID：

2.创建CrimeFragment类与实例化

CrimeFragment继承自Fragment类，并取代activity完成大部分逻辑控制工作：

/* CrimeFragment.java */

import androidx.fragment.app.Fragment;

/*未使用书中原文的Fragment(android.support.v4.app)，查到不到是因为版本过低？*/
public class CrimeFragment extends Fragment{
    private Crime mCrime;
    private EditText mTitleField;
    private Button mDateButton;
    private CheckBox mSolvedCheckBox;


    /*创建实例的生命周期方法，配置了Fragment实例，但没有生成视图*/
    @Override
    public void onCreate(Bundle savedInstanceState){//区别于activity中的onCreate为protected方法，这里为公有方法，由activity调用
        super.onCreate(savedInstanceState);
        mCrime=new Crime();
    }

    /*创建和配置视图的生命周期方法，实例化布局，然后将实例化的view返回给托管activity*/
    /*并实例化组件*/
    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container,Bundle savedInstanceState){//第二个参数为父视图，第三个参数为是否将视图添加给父视图
        /*生成视图*/
        View v=inflater.inflate(R.layout.fragment_crime,container,false);

        /*引用组件并生成监听器*/
        mTitleField=(EditText)v.findViewById(R.id.crime_title);//注意此处方法调用
        mTitleField.addTextChangedListener(new TextWatcher() {
            @Override
            public void beforeTextChanged(CharSequence charSequence,
                                          int i, int i1, int i2) {}
            
            @Override
            public void onTextChanged(CharSequence charSequence,
                                      int i, int i1, int i2) {
                mCrime.setTitle(charSequence.toString());
            }

            @Override
            public void afterTextChanged(Editable editable) {}
        });

        /*设置Button文字*/
        mDateButton=(Button)v.findViewById(R.id.crime_date);//注意此处方法调用
        mDateButton.setText(mCrime.getDate().toString());
        mDateButton.setEnabled(false);  //禁用Button按钮

        /*监听CheckBox变化*/
        mSolvedCheckBox=(CheckBox)v.findViewById(R.id.crime_solved);//注意此处方法调用
        mSolvedCheckBox.setOnCheckedChangeListener(new CompoundButton.OnCheckedChangeListener() {
            @Override
            public void onCheckedChanged(CompoundButton compoundButton, boolean b) {
                mCrime.setSolved(b);
            }
        });

        return v;
    }

}

需要注意的是，区别于activity中调用例如：

mButton = (Button)findViewById(R.id.button);

的Activity.findViewById(int)方法，其能够在后台自动调用View.findViewById(int)方法，Fragment类没有对应便捷方法，因此直接调用View.findViewById(int)方法。

第二个fragment

创建CrimeLab类

为了显示Crime列表，进行模型层的更新，新建CrimeLab类存放crime对象数组并作为单例（只有一个实例）：

/* CrimeLab.java */

public class CrimeLab {
    private static CrimeLab sCrimeLab;
    private List<Crime> mCrimes;

    public static CrimeLab get(Context context){  
        if (sCrimeLab==null)
            sCrimeLab=new CrimeLab(context);
        return sCrimeLab;
    }

    private CrimeLab(Context context){//私有构造函数 context对象见14
        mCrimes=new ArrayList<>();
    }

    public List<Crime> getCrimes(){
        return mCrimes;
    }

    public Crime getCrime(UUID id){
        for(Crime crime:mCrimes){
            if(crime.getId().equals(id))
                return crime;
        }
        return null;
    }
}

应用能在内存中存活多久，单例就能活多久，但当Android从内存中清除应用时单例对象也会消失。持久保存数据的方法见……8.7

创建CrimeListActivity类、抽象activity

由于两个activity都托管fragment，这个类的布局可以与CrimeActivity的布局通用，因此直接重命名activity_crime.xml为activity_fragment.xml，随后Android Studio自动更新（CrimeActivity.java）代码。

CrimeActivity类代码主要实现的功能是托管CrimeFragment并将其显示到activity_fragment的FrameLayout（资源ID为R.id.fragment_activity），CrimeListActivity需要实现的功能类似，代码除了实例化CrimeFragment的部分完全通用，因此把重复代码封装为抽象类：

/* SingleFragmentActivity.java */

public abstract class SingleFragmentActivity extends AppCompatActivity {//继承类

    protected abstract Fragment createFragment();//抽象方法，由子类实现并影响onCreate中实例化fragment部分

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_fragment);

        FragmentManager fm=getSupportFragmentManager();
        Fragment fragment=fm.findFragmentById(R.id.fragment_container);

        if (fragment==null){//不存在
            fragment=createFragment();//调用抽象方法新建
            fm.beginTransaction().
                add(R.id.fragment_container,fragment).
                commit();
        }
    }
}

然后修改CrimeActivity类，继承SingleFragmentActivity类，并实现createFragment()方法：

/* CrimeActivity.java */

public class CrimeActivity extends SingleFragmentActivity{
    
    /*托管到此activity的是CrimeFragment*/
    @Override
    protected Fragment createFragment(){
        return new CrimeFragment();
    }
}

可以很方便的开始编写CrimeListActivity类：

/* CrimeListActivity.java */

public class CrimeListActivity extends SingleFragmentActivity{
    
    /*托管到此activity的是CrimeListFragment*/
    @Override
    protected Fragment createFragment(){
        return new CrimeListFragment();
    }
}

创建完后在配置文件中声明它，并且配置为launcher activity：

    </intent-filter>
</activity>

RecyclerView ViewHolder N Adapter

RecyclerView是ViewGroup的子类，用CrimeListFragment来显示crime列表时需要用到，它的每一个列表项都是作为一个View子对象显示的，用户滑动屏幕切换视图时，消失在屏幕上的上一个视图会回收利用。

RecyclerView 任务是回收和定位屏幕上的View，自身不会创建视图，通过Adapter来创建。

ViewHolder任务是容纳View视图，引用着itemView。

Adapter任务是创建ViewHolder、并绑定其到模型层数据，是一个控制器对象，从模型层获取数据、然后提供给RecyclerView 显示。

使用RecyclerView

RecyclerView类来自Google支持库，需要添加RecyclerView依赖库，方法同前添加AppCompat兼容库。

//原书使用的是com.android.support.recyclerview-v7，实际编程使用了（更新的？）androidx.recyclerview.widget

在xml布局文件中添加RecyclerView视图

<!-- crime_fragment_list.xml -->
<androidx.recyclerview.widget.RecyclerView
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/crime_recycler_view"
    ...

视图与fragment关联：

/* CrimeListFragment.java */

public class CrimeListFragment extends Fragment {
    private RecyclerView mCrimeRecyclerView;  //RecyclerView作为成员变量

    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState){
        /*生成视图*/
        View view=inflater.inflate(R.layout.fragment_crime_list,container,false);

        mCrimeRecyclerView=(RecyclerView)view.findViewById(R.id.crime_recycler_view);
        /*RecyclerView视图创建后转交给LayoutManager对象*/
        mCrimeRecyclerView.setLayoutManager(new LinearLayoutManager(getActivity()));

        return view;
    }

RecyclerView摆放列表项的任务交给LayoutManager，后者还负责定义屏幕滚动行为。没有LayoutManager，RecyclerView是无法正常工作的。

更新完列表视图后，还需更新列表项视图，即每一项自己的视图层级结构：

<!-- list_item_crime.xml -->
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    ...>
    <TextView
        android:id="@+id/crime_title"
        .../>
    <TextView
        android:id="@+id/crime_date"
        .../>
</LinearLayout>

使用Adapter

在~Fragment类中定义Adapter内部类：

/* CrimeListFragment.java */
public class CrimeListFragment extends Fragment{
    ...
    private class CrimeAdapter extends RecyclerView.Adapter<CrimeHolder>{
        private List<Crime> mCrimes;

        public CrimeAdapter(List<Crime> crimes){
            mCrimes=crimes;
        }
    }
}

Adapter并没有配置完。当RecyclerView需要显示新创建的ViewHolder，或者让Crime对象和已创建的ViewHolder关联时，它会调用Adapter的方法来完成，但其并未了解具体的Crime对象。这些操作需要在CrimeAdapter类中实现并覆盖基类方法：

private class CrimeAdapter extends RecyclerView.Adapter<CrimeHolder>{
    ...
    
    /*RecyclerView需要新的ViewHolder来显示列表项*/
    @Override
    public CrimeHolder onCreateViewHolder(ViewGroup parent, int viewType) {
        LayoutInflater layoutInflater=LayoutInflater.from(getActivity());
        return new CrimeHolder(layoutInflater,parent);
    }

    @Override
    public void onBindViewHolder(CrimeHolder holder, int position) {

    }

    @Override
    public int getItemCount() {
        return mCrimes.size();
    }
}

然后把Adapter和RecyclerView关联起来，实现一个设置CrimeListFragment用户界面的方法，此方法创建CrimeAdapter并将其与RecyclerView关联：

/* CrimeListFragment.java */
public class CrimeListFragment extends Fragment{
    private CrimeAdapter mAdapter; //Adapter成员变量
    //...
    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState){
        //...
        /*创建视图时关联Adapter*/
        /*可以配置更为复杂的用户界面，在updateUI()中添加更多内容*/
        updateUI();

        return view;
    }

    private void updateUI(){
        CrimeLab crimeLab=CrimeLab.get(getActivity());
        List<Crime> crimes=crimeLab.getCrimes();
        
        /*Adapter关联模型层对象实例*/
        mAdapter=new CrimeAdapter(crimes);
        /*关联RecyclerView*/
        mCrimeRecyclerView.setAdapter(mAdapter);
    }
}

绑定列表项

此时RecyclerView视图列表项显示的是list_item_crime.xml中组件默认文本，接下来将list_item_crime组件与Java代码关联以显示实际crime数据。

由于CrimeHolder会循环使用，分开处理视图创建（创建足够多的CH后停止创建）和绑定（循环使用的过程中不断重新绑定）会更方便高效。

CrimeHolder中实例化组件，并创建bind(Crime)方法绑定Crime实例和视图

/* CrimeListFragment.java */

/*ViewHolder内部类，实例化并使用list_itme_crime布局*/
private class CrimeHolder extends RecyclerView.ViewHolder{
    private TextView mTitleTextView;
    private TextView mDateTextView;

    public CrimeHolder(LayoutInflater inflater,ViewGroup parent){
        super(inflater.inflate(R.layout.list_itme_crime,parent,false));
        mTitleTextView=(TextView)itemView.findViewById(R.id.crime_title);
        mDateTextView=(TextView)itemView.findViewById(R.id.crime_date);
    }

    /*绑定Crime实例和view*/
    /*此方法被adapter调用*/
    public void bind(Crime crime){
        mCrime=crime;
        mTitleTextView.setText(mCrime.getTitle());
        mDateTextView.setText(mCrime.getDate().toString());
    }
}

在CrimeAdapter中调用此方法：

private class CrimeAdapter extends RecyclerView.Adapter<CrimeHolder>{
    @Override
    public void onBindViewHolder(CrimeHolder holder, int position) {
        Crime crime=mCrimes.get(position);
        holder.bind(crime);
    }
}
