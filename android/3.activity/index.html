Activity

生命周期

Activity实例在其生命周期内，有运行、暂停、停止和不存在四种状态。 用户可以与当前运行状态下的activity交互，但任何时候时能有一个activity处于用户能交互的运行状态。

单击设备后退键时，系统会销毁该activity视图及其内存里的相关信息。 单击主屏幕键，Android系统会先暂停，再停止当前activity。

设备旋转

旋转设备会改变设备配置(device configuration)。 可以保存数据以应对设备旋转，覆盖方法：

protected void onSaveInstanceState(Bundle outState)

实现所有activity视图将自身状态数据保存在Bundle对象中。Bundle是存储字符串键与限定类型值之间映射关系（键-值对）的一种结构。

第二个activity

以GeoQuiz为例，添加第二个activity带来第二个用户界面供用户偷看答案。

创建第二个activity

创建新的activity至少涉及三个文件：Java类、XML布局和应用的manifest文件。可以利用Android Studio以简化新建activity过程。

第二个activity的布局组件定义在activity_cheat.xml文件中。 manifest配置文件是一个包含元数据的XML文件，用来向Android操作系统描述应用，通常在app/manifest目录下以AndroidManifest.xml命名。 应用的所有activity都必须在manifest配置文件中声明，否则操作系统无法找到他们：

<manifest xmlns:android=http://schemas.android.com/apk/res/android"
    package="com.moritz.android.geoquiz">
    
    <application
        android:allowBackup="true"
        ...>
        
        <activity android:name=".QuizActivity">
        ...
        </activity>
    
        <activity android:name=".CheatActivity">
        </activity>
    </application>
</manifest>

android:name属性值表明activity类文件位于manifest配置文件头部包属性值指定的包路径下，即com.moritz.android.geoquiz。如果属性值设置成完整的包路径com.moritz.android.geoquiz.QuizActivity，效果相同。

启动activity

使用startActivity方法：

public void startActivity(Intent intent)

activity使用该方法时，调用请求实际发送给了操作系统的ActivityManager，其负责创建Activity实例并调用onCreate(Buncle)方法。启动的Activity取决于传入的Intent参数。 intent对象是component（activity，service，broadcast receiver，content provider）用来与操作系统通信的一种媒介工具，有多个构造方法以满足不同使用需求。GeoQuiz中intent用来告诉ActivityManager该启动哪个activity，因此使用构造方法：

public Intent(Context packageContext,Class<?> cls)

Class类型参数表示该启用哪个activity，Context表示该activity在哪里。 所以mCheatButton的监听器代码如下：

mCheatButton=(Button)findViewById(R.id.cheat_button);
mCheatButton.setOnClickListener(new View.OnClickListener(){
    @Override
    public void onClick(View v){
        //启用CheatActivity
        Intent intent=new Intent(QuizActivity.this,CheatActivity.class);//创建intent实例，调用上述构造方法
        startActivity(intent);//启动
    }
});

启动前ActivityManager会确认指定的Class是否已在manifest配置文件中声明。

activity之间的数据传递

extra也是一种键值结构，可以把指定值作为extra信息附加在传入startActivity(Intent)方法的Intent发送出去。

activity的使用与管理

在各activity间往返时，操作系统层面发生如下事件： 在桌面启动器中点击应用时，启动了应用中的launcher activity，在GeoQuiz中QuizActivity就是它的launcher activity，因为manifest配置文件中被指定：

<activity android:name=".QuizActivity">
    <intent-filter>
        <action android:name="android.intent.action.MAIN"/>
        <category android:name="android.intent.category.LAUNCHER"/>
    </intent-filter>
</activity>


ActivityManager被设计成操作系统的activity管理器来负责启动应用activity的原因之一是，它维护着一个非特定应用独享的回退栈，且所有应用的activity都共享该栈。 QuizActivity实例出现在屏幕上后，用户单机CHEAT按钮，CheatActivity实例随即在QuizActivity实例上被启动，压入activity栈顶，屏幕显示CheatActivity的界面。 按后退键，则CheatActivity实例被弹出栈外。
