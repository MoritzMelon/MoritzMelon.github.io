布局和用户界面

文件app/res/layout/activity_quiz.xml定义了默认的activity布局。 组件是用户界面的构造模块，并在xml文件里定义。组件和XML元素一一对应，元素的名称就是组件的类型。各元素均有一组XML属性。 根元素（在这里是一个LinearLayout组件，继承自ViewGroup组件）组件必须指定Android XML资源文件的命名空间属性。

创建字符串资源

每个项目中，app/res/values目录下都包含一个默认字符串资源文件string.xml：

<resources>
    <string name="app_name">GeoQuiz</string>
    <string name="true_button">True</string>
</resources>

在此项目的任何XML文件中，只要引用到@string/true_button，就可以得到文本True。 一个项目也可以有多个字符串文件，只要都放在此目录下，且含有一个resources根元素和多个string子元素，应用就能找到相应的字符串资源。

获取用户界面

app/java目录是全部Java源代码的存放处，QuizAcitivity.java文件内有Activity的子类：

package com.moritz.android.geoquiz

import android.support.v7.app.AppCompatActivity;
import android.os.Bundle;

public class QuizAcitivity extends AppCompatAcitivity{

    @override
    protected void onCreate(Bundle savedInstanceState){
        super.onCreate(savedInstanceState);//调用基类构造方法，基类是一个Activity的子类，作用为兼容旧版本系统
        setContentView(R.layout.activity_quiz);
    }   
}

实例创建后，onCreate(Bundle)方法会被调用，并调用一下Activity方法获取用户界面：

public void setContentView(int layoutResID)

资源与资源ID

布局是一种资源。资源是非代码形式的内容。项目的所有资源都在app/res的子目录下，例如布局资源文件存放在res/layout/，字符串资源文件存放在res/values/。在代码中获取相应的资源的方法则是使用资源ID。 目录app/build/generated/source/debug下的R.java文件是项目编译过程中自动生成的、包含项目当前所有资源ID的文件，部分截取如下：

public final class R{
    //...
    public static final class layout{
        //...
        public static final int activity_quiz\0x7f030017;//xml布局资源ID
    }
    public static final class string{
        public static final int app_name=0x7f0a0010;
        public static final int true_button=0x7f0a0012;
    }
}

activity_quiz.xml布局的资源ID为R.layout.activity_quiz，是R的内部类layout里的一个整型常量名。 布局文件中的组件并不会默认生成资源ID，如果需要在代码中需要与组件交互，则为该组件生成资源ID即可，在定义组件时为其添加android:id属性：

<LinearLayout...>
    ...
    <Button
        android:id="@+id/true_button"
        android:text:"@string/true_button"
        ...

android:id属性值前面有一个+标志，意为创建资源ID，而对字符串资源只是做引用。

组件的实际应用

引用视图对象

在activity_quiz.xml文件给Button生成资源ID后，就可以在QuizActicity中获取：

import android.widget.Button

public class QuizActivity extends AppCompatActivity{
    private Button mTrueButton;//m前缀，约定俗成的成员变量名称
    
    @override
    protected void onCreate(Bundle savedInstanceState){
        //...
        mTrueButton=(Button)findViewById(R.id.true_button);
    }
}

其中，调用了Activity方法：

public View findViewById(int id)

以组件资源id为参数获取一个视图对象，并类型转换为Button。
