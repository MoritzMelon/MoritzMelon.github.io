GeoQuiz与MVC设计模式

Android应用基于模型-视图-控制器(Model-View-Controller,MVC)的架构模式进行设计，任何对象都属于以下的其中一种： 1. 模型对象：存储应用的数据和业务逻辑，例如GeoQuiz应用的Question类 2. 视图对象：例如Button，TextView 3. 控制器对象：含有应用的逻辑单元，联系视图对象与模型对象，通常是Activity、Fragment或Service的子类。

在GeoQuiz小程序中，创建模型对象Question来储存问题文本和问题答案。

创建新类

新建Question类，用int类型变量保存问题文本的字符串资源ID，并对新增的两个变量编写getter和setter方法，可以由Android Studio自动生成。

public class Question{
    private int mTextResId;
    private boolean mAnswerTrue;
    
    public Question(int textResId,boolean answerTrue){
        mTextResId=TextResId;
        mAnswerTrue=AnswerTrue;
    }
    
    public int getTextResId(){
        return mTextResId;
    }
    
    public void setTextResId(int textResId){
        mTextResId=TextResId;
    }   
    //...
}

更新视图层

现在用Question类来保存问题，就不需要直接在视图层的TextView中硬编码文本，可以增加组件的资源ID以便在QuizActivity代码中设置组件显示的文字：

<LinearLayout...>
    <TextView
        android:id="@+id/question_text_view"
        ...

问题的字符串在string.xml文件中更新：

<string name="question_australia">Canberra is the capital of Australia.</string>
<string name="question_china">Beijing is the capital of China.</string>
...

其中可以使用转义字符来得到相应符号。

添加图标资源

项目中所有的图片资源都会随应用安装在设备里，Android操作系统自动为不同的设备提供最佳匹配，并且可以自动进行图片的缩放适配。 任何添加到res/drawable目录中、后缀名为.jpg .png .gif的文件都会自动获得资源ID，但文件名必须是小写字母且不能有空格符号。 而在XML资源文件中，以@string/开头的定义是引用字符串资源，而以@drawable/开头的定义是引用drawable资源：

    android:id="@+id/next_button"
    android:drawableRight="@drawable/arrow_right"
    android:drawablePadding="4dp"
