2.2 标识符

规则：由26个英文字符大小写、0-9、_或$组成，不可以以数字开头，严格区分大小写，不包含空格

规范：

  • 包名：多单词组成时所有字母都小写，xxxyyyzzz
  • 类名、接口名：多单词组成时，所有单词的首字母大写，XxxYyyZzz
  • 变量名、方法名：多单词组成时，第一个单词首字母小写，第二个单词开始每个单词首字母大写，xxxYyyZzz

规范不一定要遵守，但遵守有利于增强代码可读性

整型
  • Java各整数类型有固定的表数范围和字段长度，不受具体OS的影响，以保证Java程序的可移植性
  • byte类型占用1 B=8 bit的存储空间，范围-128~127，和short类型主要用于底层文件处理或控制占用存储空间量的大数组等场合
  • 整型常量默认为 int型，声明long型常量须后加l或L
    e.g.long l1=123456789L;
  • 十六进制有前缀0X或0x，八进制有易混淆的前缀0，Java7开始有二进制前缀0b或0B，并且可以在数字之间加入下划线以表数位划分
  • 没有unsigned形式


浮点类型
  • 浮点型常量默认为double型，声明float型常量，须后加f或F
  • 可使用十六进制表示浮点数值，用p代替e来表示指数，尾数采用十六进制，指数采用十进制，基数为2（而不是10），e.g.double d=0x1.0p-3;，其中$0.125=2^{-3}$
  • 浮点数值计算遵循IEEE 754规范，用于表示溢出和出错情况的三个特殊浮点数值为：

    正无穷大（e.g.一个正整数除以0的结果）

    负无穷大

    NaN（不是一个数字，e.g.0/0、负数平方根的结果）

    并且有相应常量表示
  • 计算中不允许有任何舍入误差则该使用BigDecimal类，例如$2.0-1.1$的结果打印出来则是$0.899999999999$


字符型（2 byte）

char c='\u000a';//表示'\n'

但\u0022+\u0022表示的并不是字符串"+"，转移序列在解析前得到处理，即得到的结果为""+""，也就是一个空串。

另外，注释中也会出现\u的转义

  • ASCII 码
    规定了128个字符的编码，只占用了一个字节的后面7位，最前面的1位统一规定为0，不能表示所有字符，并且相同的编码表示的字符不一样。
  • Unicode
    将世界上所有的符号都纳入其中，每一个符号都给予一个独一无二的编码，没有乱码的问题
    缺点：1.Unicode 只是一个符号集，它只规定了符号的二进制代码，却没有规定这个二进制代码应该如何存储
    2.无法区别 Unicode 和 ASCII：计算机无法区分三个字节表示一个符号还是分别表示三个符号
  • UTF-8 8-bit Unicode Transformation Format
    UTF-8 是在互联网上使用最广的一种 Unicode 的实现方式。UTF-8是一种针对Unicode的可变长度字符编码（1-6 个字节），也是一种前缀码。它可以用来表示Unicode标准中的任何字符，且其编码中的第一个字节仍与ASCII兼容

4.进制和原码、反码、补码

二进制以0b或0B开头，八进制以0开头，十六进制以0x或0X开头且不区分字母大小写。

正数的原码、反码、补码相同，最高位的符号位为0，负数：

  • 原码：正数最高位符号位改为1
  • 反码：各位除符号位取反
  • 补码：反码+1，负数在底层的存储格式，-1补码为11111111，-128补码为10000000

11 1111 1111 1111 1111 1111 1111 1110 00|01

结果为-8：

1111 1111 1111 1111 1111 1111 1111 1000

而进行>>>运算时，最高位用0来补位

00 1111 1111 1111 1111 1111 1111 1110 00|01

2.6 数组

1. 初始化和定义

使用关键字new创建数组对象，声明数组时不能指定其长度(数组中元素的数)

有两种初始化数组的方式，一是动态初始化：数组声明，为数组元素分配空间与赋值的操作分开进行

int arr[5];  //非法！！！
int[] arr = new int[3];
arr[0] = 3;
arr[1] = 9;
arr[2] = 8;

静态初始化：初始化和赋值同时进行

int a1[] = new int[]{3, 9, 8};
int a3[] = {3,9,8};
int[] a2;
a2 = new int[]{3,9,8};
MyDate dates[] = {
        new MyDate(22, 7, 1964), //再new
        new MyDate(1, 1, 2000),
        new MyDate(22, 12, 1964)
}；
MyDate d1[]=new MyDate[3];

数组长度可以为变量（区别于C++）

数组属引用类型，数组型数据是对象(object)，数组中的每个元素相当于该对象的成员变量，因此数组一经分配空间，其中的每个元素也被按照成员变量同样的方式被隐式初始化

其中，基本数据类型的默认初始化值各有不同，引用数据类型而言，默认初始化值为null


数组一旦初始化，其长度是不可变的，e.g.array.length 指明数组的长度(元素个数)


与C++不同，支持用数组给数组赋值：

int[] arr1,arr2[];
arr1 = new int[5];
arr2 = new int[3][2];
arr2[0]=arr1; //合法
System.out.println(arr1);     //打印出的是地址
System.out.println(arr2[0]);  //输出结果和上一行一样

这种做法实际上是把arr1指向堆空间地址值赋给arr2[0]，arr1元素的值改变会导致arr[2]的值同样改变，本质上指向的是同一段空间

如果要真正实现数组的复制拷贝，可以调用System.arraycopy()


2. 默认初始化

  1. 整数类型byte, short, int, long，值为0
  2. 浮点类型float, double，值为0.0
  3. char默认值为空格
  4. boolean默认值为false
  5. 引用类型变量（e.g.String）值为null（打印结果也为null）


3. 多维数组

以二维数组为例，即数组的数组

动态初始化的两种格式：

int[][] arr1 = new int[3][2];

arr1的二维数组中有3个一维数组，分别叫arr1[0],arr1[1],arr1[2]，每个一维数组中有两个元素

可以用arr1.length来获取数组大小（行数），在此例下结果为3（行）

int[][] arr2 = new int[3][];
arr2[0] = new int[3];
arr2[1] = new int[1];
arr2[2] = new int[2];

arr2的二维数组中有3个一维数组,每个一维数组都是默认初始化值null，对这个三个一维数组分别进行初始化

第一步声明时：在内存中，变量名arr2压入栈，并在堆中new三个值为null的空间，把首元素地址赋值给arr2；

对一维数组初始化时：在堆中new新的空间，并把新空间的首元素地址赋值给arr2[0]

一些注意事项：

int[][] arr3 = new int[][3]; //非法！没有指定（元素为数组的）数组的维度
int[] arr5,arr6[];           //特殊写法，arr5为一维数组，arr6为二维数组

静态初始化：

int[][] arr4 = new int[][]{{1},{2,3},{4,5,6}};


4. 数组的内存结构

Java内存的基本结构中有：

  • 栈stack：存放局部变量、对象的引用
  • 堆heap：new出来的数组、对象等
  • 方法区：类、包、定义的方法，其中包含常量池
  • 静态域：static修饰的静态变量

以以下数组定义过程为例：

int a[] = new int[4];  //动态初始化

新定义的变量名a压入栈中，在堆中开辟了一段空间（并默认初始化为0），把空间开始的地址值（四位十六进制数）赋给变量a。静态初始化同理。

需要注意类的初始化：

MyDate date[]=new MyDate[3];
date[0] = new MyDate(1,1,2000); //此处new不可省略

变量名date压入栈中并把数组首元素地址值赋给它，但数组元素值仍然是null，需要开辟一个新的空间存放类的成员变量，并把新的空间的首地址值赋给date[0]

多维数组内存结构见上方有关多维数组的描述


5.数组常见异常exception

以下常见异常在编译阶段不会报错

  • sort()：用来对array进行排序
  • binarySearch()：在排好序的array中寻找元素
  • 另外，System中有方法arraycopy()，可以实现array的复制。
