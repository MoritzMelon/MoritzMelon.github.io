Lesson 4: Inheritance
  • Jul 24, 2019

一、类 子类 超类

1 定义子类

关键字extends表示继承：

public class Manager extends Employee{
    //...
}

不同于C++的多种继承方式，Java所有的继承都是公有继承。

被继承的Employee类成为超类(superclass)、基类(base class)、父类(parent class)。

新类Manager为子类(subclass)、派生类(derived class)、孩子类(child class)。

2 覆盖方法

超类的方法若在子类中内容不同，则需要提供一个新的方法来覆盖(override)超类的方法。

子类方法不能直接访问超类的私有域。

在子类Manager中重新定义一个方法，使用超类同名方法时用关键字super：

public double getSalary(){
    return super.getSalary()+bonus;
}

super不是对象的引用（不同于this），只是一个特殊关键字。

注意：C++中调用超类方法的语法为Employee::getSalary()

3 子类构造器

子类构造器的第一条语句使用super关键字调用超类构造器：

public Manager(String name,double salary){
    super(name,salary);
    bonus=0;
}

如果不显式调用超类构造器，则自动调用超类默认构造器。

4 继承层次

由一个公共超类派生出来的所有类的集合为继承层次(inheritance hierarchy)。

继承层次中，从某个特定的类到其祖先的路径为该类的继承连(inheritance chain)。

Java不同于C++，不支持多继承。Java的多继承功能实现方式与接口相关。

5 多态

Java不需要将方法声明为(C++)虚拟方法，动态绑定是默认处理方式。

可以将一个子类对象赋给超类变量。例如，Employee型对象引用，既可以引用Employee型（超类）对象，也可以引用Manager型（子类）对象，并且虚拟机能根据实际引用的对象类型调用相应方法：

Manager boss = new Manager("Carl",8000);
Employee[] staff = new Employee[3];
staff[0]=boss;
for(Employee e:staff)
    System.out.println(e.getSalary());

这种一个对象变量可以指示多种实际类型的现象为多态(polymorphism)，在运行时能够自动地选择调用哪个方法的现象称为动态绑定(dynamic binding)。

如果不希望一个方法具有虚拟特征，可以标记为finale。

此时，超类变量只能调用超类方法，不可以调用子类拓展出来的方法：

boss.setBonus(3000);     //ok
staff[0].setBonus(5000); //error!

也不可以将一个超类对象赋给子类变量：

Manager m=staff[2];     //error

7 final类和方法

不允许扩展的类为final类：

public final class Executive extends Manager{
    //...
}

由Manager扩展出来的Executive类不能拥有子类。

类中被声明为final的方法不会被子类覆盖。

final类的所有方法自动成为final方法，但域不是final。
