Lesson 3: Object and Class
  • Jul 18, 2019

学习面向对象内容的三条主线：Java类及类的成员，面向对象的三大特征，其它关键字

3.1 面向对象程序设计-OPP概述

Object Oriented Programming 三大特征：封装 (Encapsulation)，继承 (Inheritance)，多态 (Polymorphism)

3.1.1 类

类(class)是构造对象的模板或蓝图。由类构造(construct)对象的过程称为创建类的实例(instance)。

对象中的数据称为实例域(instance field)，操纵数据的过程称为方法(method)。每个对象的特定实例域值的集合就是当前的状态(state)。

3.1.2 对象

对象三个主要特征：行为(behavior)，状态(state)，标识(identity)

3.1.3 识别类

类通常为分析问题的过程中出现的名词，而方法对应动词

3.1.4 类之间的关系

类之间最常见关系有：

  1. 依赖(dependence) uses-a ：一个类的方法操纵另一个类
  2. 聚合(aggregation) has-a ：一个对象包含另一个对象，即子对象
  3. 继承(inheritance) is-a ：拓展类

可以采用UML(Unified Modeling Language，统一建模语言)绘制类图


3.2 使用预定义类

3.2.1 对象与对象变量，构造器

Java（和C++）使用构造器(constructor)构造新实例，是一种特殊的方法，其方法名与类名相同。

如果要构造一个对象，需要在构造器前加上new操作符：

new Date()

可以把这个对象传递给一个方法：

System.out.print(new Date());

可以应用对象的一个方法：

String s=new Date().toString();

可以把对象 存放在一个对象变量中：

Date birthday=new Date();
Date deadline;           //deadline doesn't refer to any object
/*等同于C++语句：
Date* deadlint;
*/

最后一句定义了对象变量deadline，但它不是一个对象，实际上也没有引用对象，所以不可以将任何Date方法应用于deadline。

deadline=birthday;    //引用同一个对象
deadline.toString();  //现在可以应用方法了

一个对象变量并没有实际包含一个对象，而仅仅引用一个对象，在Java中，任何对象变量的值都是对存储在另一个地方的一个对象的引用。new操作符返回值也是一个引用。

Java所有对象都存储在堆中，当一个对象包含另一个对象变量时，实则这个变量包含了指向另一个堆对象的指针，直接用对象变量名赋值得到的是同一个对象，如果要进行复制，必须使用clone方法获得对象的完整拷贝。

null

可以把对象显示地赋值为null，如果将一个方法应用于其上，会产生运行时错误。

局部变量不会自动初始化为null，需要设置值。


3.2.2 LocalDate类

纪元(epoch)：UTC(Coordianted Universal Time)时间1970年1月1日00:00:00。

静态工厂方法(factory method)


3.2.3 更改器方法与访问器方法

更改器方法(mutator method)：调用后状态对象改变

访问器方法(accessor method)：只访问而不修改对象



3.3 用户自定义类

主力类(workhouse class)，通常没有main方法，却有自己的实例域和实例方法。一个完整的程序由若干类组合在一起，其中只有一个类有main方法。

3.3.1 Employee类

定义：

class Employee{
    //instance field
    private String name;
    private double salary;
    private LocalDate hireDay;//包含其他对象
    
    //constructor 构造器
    public Employee(String n,double s,int year,int month,int day){
        //....
    }
    
    //methods
    public String getName(){
        return name;
    }
    //...
}   //没有分号！

使用：

//目录： EmployeeTest/EmployeeTest.java 源文件名与唯一的public类名相匹配
import java.time.*;
public class EmployeeTest{
    public static void man(String[] args){
        Employee[] staff=new Employee[3];
        staff[0]=new Employee("Tom",1333,1983,3,3);
    }    
}

class Employee{
    //...
}

3.3.2 多个源文件使用

可以将两个类分别存放在单独的源文件中，编译方法有两种：

1.使用通配符调用Java编译器，所有与通配符匹配的源文件都将被编译成类文件

javac Employee*.java

2.直接编译EmployeeTest.java文件

javac EmployeeTest.java

编译器发现其使用了Employee类时会查找名为Employee.class文件，如果没找到就会自动搜索Employee.java并对其进行编译。如果.java文件版本比已有的.class文件更新，也会重新编译这个文件。


3.3.3 构造器

构造器总是伴随着new操作符的执行被调用（注意与C++稍有不同）

Employee number03("Tom",1333,1983,3,3); //错误！
Employee number03=new Employee("Tom",1333,1983,3,3);

每个类可以有多个构造器，构造器没有返回值且与类同名。

注意所有方法中不要命名与实例域同名的变量。


3.3.4 隐式参数与显式参数

以Employee类中的一个方法为例：

public String getName(){
        return name;
}

这个方法有两个参数：

第一个参数成为隐式(implicit)参数，是出现在方法名前面的Employee类对象，不出现在方发声明中在方法中用关键字this表示。

第二个参数位于方法名后的括号中，是显式(explicit)参数。

上述方法可以如下编写：

public String getName(){
        return this.name;  //运用this
}

Java中所有方法都必须在类内部定义，哪个方法是否设置为内联(inline)是虚拟机的任务。


3.3.5 封装的优点

如果没有任何方法可以改变某个实例域，则称为只读域。

为一个私有数据域提供一个公有的域访问器方法、一个公有的域更改器方法，比直接提供一个公有数据域要复杂，但有许多好处：

  1. 可以改变内部实现，不影响类外部的代码
  2. 更改器方法可以执行错误检查
  3. 域值出现错误只需调试方法

注意！！：不要编写返回引用可变对象的对象器方法，例如，若Employee类中定义了如下访问器方法：

public Date getHireDay(){
    return hireDay;  //bad!!
}

又Date类有更改器方法setTime，所以Date对象可变，因为该方法返回该对象的引用，并不是对象的一个克隆(clone)，应修改代码为如下所示：

public Date getHireDay(){
    return (Date)hireDay.clone();
}


3.3.6 基于类的访问权限

一个方法可以访问所属类的所有对象的私有数据。


3.3.7 final实例域

构件对象时必须初始化被定义为final的实例域，后续操作中不能再修改，语法例：

class Employee{
    private final String name;
}

final修饰符大都应用于基本(primitive)类型域，或不可变(immutable)类的域（即，类的每个方法都不会改变其对象，例如String类）。

对于可变类中，例如：

private final StringBuilder name;

在构造器中初始化为：

name = new StringBuilder();

final表示name对象引用不会改变，但该对象可以更改（类似C++中的常指针）



3.4 静态域和静态方法

1 静态域

每个类中只有一个定义为static的域，这个类的所有实例共享同一个静态域。即使没有对象，静态域也存在，它属于类，也被称为类域。在类体中进行赋值：

class Employee{
    private static int next=1;
}

2 静态常量

例如Math类中定义了一个静态常量π：

public class Math{
    public static final double PI = 3.14159265358979323846;
}

并在程序中采用Math.PI获得这个常量。

另一个唱见静态常量是System.out。


注：setOut可以改变fianl变量System.out的值，但是一个本地方法，并非用Java实现。


3 静态方法

静态方法不能对对象实施操作，没有隐式参数（this参数），例如Math.pow(x,a)，运算时不使用任何Math对象。所以不需要构造对象就可以调用静态方法。

静态方法可以访问自身类中的静态域，例如：

public static int getNext(){
    return next;
}

可以通过类名调用：

int x=Employee.getNext();

注：C++中用类名调用静态方法时，使用的是::运算符，例如Math::PI

4 工厂方法

详见《 Java核心技术·卷 I（原书第10版）》P115

5 main方法

main方法不对任何对象进行操作：

public class test{
    public static void main(String[] args){
        //...
    }
}

每一个类都可以有一个main方法，可以为做对类进行单元测试的技巧。

例如在Employee类中添加一个main方法，并且用语句java Employee来独立测试这个类，然后再执行语句java test运行程序，此时Employee类的main方法不会执行。



3.5 方法参数

将参数传递给方法/函数时：

  • 按值调用(call by value)：方法接收值
  • 按引用调用(call by reference)：方法接收变量地址

“按……调用”(call by)专业术语用来描述各种程序设计语言中传参方式。

Java总是按值调用，方法得到的是所有参数值的一个拷贝。

当对象引用作为参数时：

public void changeSalary(Employee x){
    x.salary+=100;
}

调用：

tom = new Employee();
changSalary(tom);

改变了tom对象，过程如下：

  1. x初始化为tom值的拷贝，是一个对象的引用
  2. 方法引用于该对象，即x和tom共同引用的Employee对象
  3. 方法结束，x不再使用，tom继续引用同一个值改变的对象



3.6 对象构造

初始化数据域的方法过程：

  1. 默认值（0，false和null）
  2. 按照类声明中的次序，依次执行域初始化语句和初始化块
  3. 执行构造器调用的另一个构造器的主体
  4. 执行构造器主体

1 重载

多个方法同名、不同参数，则产生了重载(overloading)。编译器匹配挑选相应的重载方法，如果找不到匹配的参数则产生编译时错误，这个过程成为重载解析(overloading resolution)。

Java允许重载任何方法。要完整地描述一个方法需要指出方法名即参数类型，即方法的签名(signature)，例如String类四个公有方法的签名：

  • indexOf(int)
  • indexOf(int,int)
  • indexOf(String)
  • indexOf(String,int)

其中返回类型不是签名的一部分。

2 默认域初始化

默认赋值有：数值为0，布尔值为false，对象引用为null

构造器不显示地给域赋初值，则自动地赋为默认值，但方法中的局部变量必须被明确地初始化。

3 无参数构造器

例如：

public Employee(){
    name="";
    salary=0;
    hireDay=LocalDate.now();
}

若没有编写构造器，则系统会提供一个无参数构造器，将所有实例域设置为默认值。

若类中至少有一个构造器，但没有无参数构造器，则构造对象时不提供参数会被视为不合法。

希望所有域被赋予默认值，可以编写函数体为空的无参构造器：

public Employee(){
}

4 显式域初始化

可以在类定义中，直接将一个值赋给任何域：

class Employee{
    private String name="";
    private static int next;
    private int id=assignId();
    
    private static int assignId(){
        int r=next;
        next++;
        return r;
    }
}

先执行赋值操作，再执行构造器。并且可以调用方法进行初始化。

Java没有C++中的初始化器列表语法。

5 参数名

为了增加代码可读性，编写构造函数时可以：

public Employee(String aName,double aSalary){
    name = aName;
    salary = aSalary;
}

或者运用this隐式参数绕开同名变量的屏蔽：

public Employee(String name,double salary){
    this.name = name;
    this.salary = salary;
}

6 构造器调用构造器

采用this关键字，重复运用公共的构造器代码部分：

public Employee(double s){
    //调用 Employee(String,double)
    this("Employee #"+next,s);
    next++;
}

C++中不允许构造器调用另一个构造器。

7 初始化块

初始化块(initialization block)：类的声明中可以包含多个代码块，只要构造对象，这些块就会被执行。

例如：

class Employee{
    private static int next;
    private int id;
    private String name;
    private double salary;
    
    {
        id=next;
        next++;
    }
    
    public Employee(){
        //....
    }
    //....
}

建议将初始化块放在域定义之后。

也可以有静态的初始化域，需要标记关键字static：

static
{
    Random generator=new Random();
    next=generator.nextInt(10000);
}

8 析构与finalize方法

由于Java有自动的垃圾回收器，所以不支持析构器。

在对象使用了内存之外的资源（如文件，或者系统资源的另一个对象的句柄），则需要对资源进行回收和再利用。

为一个类添加finalize方法，将在垃圾回收器清除对象之前调用。

详细操作暂不学习。



3.7 包

标准Java类库分布在多个包(package)中。Java包具有层次结构，可以使用嵌套层次包。

所有标准Java包都处于java和javax包层级中。

不同包中的类，即使同名也不会产生冲突，保证类名的唯一性。

1 类的导入

一个类可以访问所属包中的所有类，以及其他包中的公有类(public class)。

访问另一个包中公有类的两种方式

第一种，在类名前添加完整的包名：

java.time.LocalDate today = java.time.LocalDate.now();

第二种，使用import语句，位于源文件顶部、package语句后面，例如：

导入java.util包中所有类：

import java.util.*;

其中，只能用星号*导入一个包，不能使用例如java.*导入以java为前缀的包。

导入特定类：

import java.time.LocalDate;

若使用了多个导入包中都包含的类时，可以增加一个特定的import语句来解决：

import java.util.*;
import java.sql.*;
import java.util.Date;

若需要使用类名冲突的多个类，则采用最原始的方式，在类名前加上包名：

java.util.Date day1;
java.sql.Date day2;

2 静态导入

导入System类的静态方法和静态域：

import static java.lang.System.*;
out.println("Hello!\n");//可以直接使用

导入特定方法或域：

import static java.lang.System.out;

3 将类放入包中

将包名放在源文件开头、定义类代码之前，可以把类放在特定包中，否则放入默认包(default package)：

package com.corejava;

public class Employee{
    //...
}

然后包中文件放到与完整包名匹配的子目录中。例如，com.corejava包中的所有源文件（.java文件和.class文件）放置在目录com\corejava中。

4 包作用域

如果不把一个类定义为公有类，那么同一个包中的其他类都可以访问这个类。

如果变量不显式地标记为private，那么默认为包可见。



3.8 类路径

类路径必须与包名匹配。

类文件可以存储在JAR（Java归档）文件中，其中使用ZIP格式组织文件和子目录。

使类被多个程序共享和设置类路径的具体方法见书。



3.9 文档注释

javadoc工具可以由源文件生成一个HTML文档。

javadoc实用程序(utility)从几个特征中抽取信息：

  1. 包
  2. 公有类与接口
  3. 公有的和受保护的构造器、方法
  4. 公有的和受保护的域

注释以/**....*/为界定，中间包含自由格式文本(free-form text)，标记由@开始。



3.10 类设计技巧

OOP设计中需要注意：

  1. 数据处理：不破坏封装性，必须进行初始化，合理编写域访问器和域更改器
  2. 类设计：分解类的职责，类名和方法名提现职责，优先使用不可变类
