2.1 关键字

2.1.1 关键字

关键字均为小写，用做专门用途，类别包括：

  • 用于定义数据类型：class，interface，byte，boolean
  • 用于定义数据类型值：true，false，null
  • 用于定义流程控制
  • 用于定义访问权限修饰符
  • 用于定义类，函数，变量修饰符：abstract，final，static，synchronized
  • 用于定义类与类之间关系：extends，implements
  • 用于定义建立实例及引用实例，判断实例：new，thi，super，instanceof
  • 用于异常处理：try，catch，finally，throw，throws
  • 用于包：package，import
  • 其他修饰符：native，strictfp，transient，volatile，assert

2.1.2 保留字

现有Java版本尚未使用，但以后版本可能会作为关键字使用

byValue cast future generic inner operator outer rest var goto const


2.2 标识符

规则：由26个英文字符大小写、0-9、_或$组成，不可以以数字开头，严格区分大小写，不包含空格

规范：

  • 包名：多单词组成时所有字母都小写，xxxyyyzzz
  • 类名、接口名：多单词组成时，所有单词的首字母大写，XxxYyyZzz
  • 变量名、方法名：多单词组成时，第一个单词首字母小写，第二个单词开始每个单词首字母大写，xxxYyyZzz
  • 常量名：所有字母都大写。多单词时每个单词用下划线连接，XXX_YYY_ZZZ

规范不一定要遵守，但遵守有利于增强代码可读性

2.3.1 基本数据类型

基本数据类型 数值型>整数类型 byte,short,int,long
数值型>浮点类型 float,double
字符型 char
布尔型 boolean
引用数据类型 类（字符串） class
数组 [ ]
接口 interface
补充：变量的分类，学到面向对象时再回顾并理解 -2019.4.28

  • Java各整数类型有固定的表数范围和字段长度，不受具体OS的影响，以保证Java程序的可移植性
  • byte类型占用1 B=8 bit的存储空间，范围-128~127
  • 整型常量默认为 int型，声明long型常量须后加l或L
    eg.long l1=123456789L;
  • 浮点型常量默认为double型，声明float型常量，须后加f或F
  • boolean类型数据只允许取值true和false，无null，不可以0或非 0 的整数替代false和true，这点和C语言不同

字符型（2 byte）

可以表示一个英文/中文/标点符号等字符

char c='我';

允许转义字符，也可以直接用Unicode值表示字符常量'\uXXXX'，用十六进制数表示，并且可运算

char c='\u000a';//表示'\n'
  • ASCII 码
    规定了128个字符的编码，只占用了一个字节的后面7位，最前面的1位统一规定为0，不能表示所有字符，并且相同的编码表示的字符不一样。
  • Unicode
    将世界上所有的符号都纳入其中，每一个符号都给予一个独一无二的编码，没有乱码的问题
    缺点：1.Unicode 只是一个符号集，它只规定了符号的二进制代码，却没有规定这个二进制代码应该如何存储
    2.无法区别 Unicode 和 ASCII：计算机无法区分三个字节表示一个符号还是分别表示三个符号
  • UTF-8 8-bit Unicode Transformation Format
    UTF-8 是在互联网上使用最广的一种 Unicode 的实现方式。UTF-8是一种针对Unicode的可变长度字符编码（1-6 个字节），也是一种前缀码。它可以用来表示Unicode标准中的任何字符，且其编码中的第一个字节仍与ASCII兼容
