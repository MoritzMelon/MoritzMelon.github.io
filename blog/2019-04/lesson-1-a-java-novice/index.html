配置编译环境

环境变量设置

下载eclipse和JDK，但直接解压软件包是不行的，需要设置环境变量

  1. 变量名JAVA_HOME：C:\Program Files (x86)\Java\jdk1.6.0_45，即jdk的具体安装路径
  2. CLASSPATH：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;，开头.的含义是当前目录下
  3. Path最后加上;%JAVA_HOME%\bin
  4. 检测：windows运行命令行，输入java -version检测是否配置成功


问题

  1. eclipse no java virtual machine was found after searching the following locations
    解决：环境变量配置失败，重新配置
  2. eclipse failed to load the jni shared library
    解决：参考网址，原因应该是安装了64位的JDK，eclipse是32位的，变量JAVA_HOME的内容改成32位JDK即解决


*常用的DOS命令

  • dir：directory，列出当前目录下的文件以及文件夹
  • md：Make Directory，创建目录
  • rd：Remove Directory，删除目录
  • cd：Change Directory，进入指定目录
  • cd..：退回到上一级目录
  • cd\：退回到根目录
  • del：delete，删除文件
  • ren：rename，重命名
  • exit：退出 dos 命令行



Java特点

与C++对比：舍弃指针（以引用取代）、运算符重载、多重继承等特点，增加垃圾回收功能*(interesting!)*，（JDK 1.5引入）泛型编程（Generic Programming）、类型安全的枚举、不定长参数和自动装/拆箱。

  • 强制面向对象
  • 分布式：Java语言支持Internet应用的开发，在基本的Java应用编程接口中有一个网络应用编程接口（java net），它提供了用于网络应用编程的类库，包括URL、URLConnection、Socket、ServerSocket等。Java的RMI（远程方法激活）机制也是开发分布式应用的重要手段。目前未能理解该段内容含义。 -2019.4.27

  • 健壮的！：强类型机制、异常处理、垃圾的自动收集（Garbage Collection，两种核心机制之一，程序运行过程中自动进行，无法被人为精确控制和干预）等

  • 解释型，跨平台性：字节码格式和Java虚拟机Java Virtual Machine（核心机制之一）

  • Java SE(Java Standard Edition)标准版：支持面向桌面级应用（如Windows下的应用程序）的Java平台

  • JDK中包含了JRE，JRE中包含了JVM，JVM运行于操作系统之上，它依赖于操作系统



HelloWorld

命令行编写并运行Java程序

新建HelloWorld.java文件，在其中编写如下程序

/**
这是我的第一个Java程序
@author Melon
@version v1.0
*/
class HelloWorld{//类声明
    public static void main(String[] args){//main方法，程序的入口，格式固定
        System.out.println("Hello world!"+'\t'+"Hello Java!");//用+连接
    }
}

一个源文件中，最多之能有一个类声明为public，且类名与原文件名相同

命令行中执行如下指令，将源文件编译为.class字节码文件

javac HelloWorld.java

执行如下命令以运行程序

java HelloWorld

在控制台窗口输出一行”Hello world!”

特有的文档注释

格式：

/**
*@author  指定java程序的作者
*@version  指定源文件的版本
*@param   方法的参数说明信息
*/

注释内容可以被JDK提供的工具javadoc 所解析，生成一套以网页文件形式体现的该程序的说明文档，命令如下

javadoc -d FILENAME -author -version HelloWorld.java

在FILENAME文件夹中打开index.html可以更直观查看到程序中使用的类、方法等，在程序更长更复杂的情况下提供一个便捷的审阅方法。
